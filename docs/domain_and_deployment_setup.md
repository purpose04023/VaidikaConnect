# VaidikaConnect Domain Registration & Deployment Planner

This guide outlines the end-to-end steps required to purchase, configure, secure, and launch the VaidikaConnect application in production.

---

## 1. Domain Purchase & Registration

You can register your primary domain (e.g., `vaidikaconnect.com`) through any major domain registrar:
- **Registrar Recommendations:** Namecheap, GoDaddy, Squarespace Domains, or Cloudflare Registrar.
- **Process:**
  1. Search for `vaidikaconnect` (or matching TLDs like `.in`, `.org`, `.co`).
  2. Purchase the domain and enable **Whois Privacy Protection** (usually free) to secure owner details.

---

## 2. DNS & Custom Subdomain Configuration

If using **Vercel** (recommended for Next.js 15) to host the frontend:
1. Log in to your Vercel Dashboard, select the project, and go to **Settings > Domains**.
2. Add your custom domain: `vaidikaconnect.com` and `www.vaidikaconnect.com`.
3. Vercel will provide the required DNS records:
   - **Apex Domain (`@`):** Add an `A` record pointing to `76.76.21.21`.
   - **WWW Subdomain (`www`):** Add a `CNAME` record pointing to `cname.vercel-dns.com`.
4. Allow 10–30 minutes for DNS propagation. SSL certificates will be auto-generated by Let's Encrypt through Vercel.

---

## 3. Production Supabase Configuration

To connect your live database securely:
1. Go to **Supabase Dashboard > settings > API**.
2. Copy your **Project URL** and **Anon Public Key**.
3. Go to **Settings > Database** and copy the Connection String for environment variables.
4. **Row Level Security (RLS):**
   - Ensure RLS is active on all tables.
   - Run the RLS enforcer script on `global_settings`:
     ```sql
     ALTER TABLE public.global_settings ENABLE ROW LEVEL SECURITY;
     ```

---

## 4. WhatsApp Business API (Twilio / Meta Portal)

To activate automatic booking confirmations:
1. Register an account on the **Meta Developer Portal** (developers.facebook.com) or **Twilio**.
2. Create a Business App and request access to the **WhatsApp Business Platform**.
3. Set up your WhatsApp templates (e.g. customized Muhurtha / Pandit assignments).
4. Update your production environment variables:
   - `WHATSAPP_API_TOKEN`: Your API authentication bearer token.
   - `WHATSAPP_PHONE_NUMBER_ID`: The sender phone number ID from Meta.
   - `WHATSAPP_RECIPIENT`: Destination number (fallback).

---

NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
